# Playwright Flow Skeleton

## Goal

From the same trace, emit a Playwright script skeleton that replays the
recorded flow by resolving each step's ranked locators in order. It is a
**skeleton, not a test**: assertions are suggested as comments, never asserted.
The recorder does not know intent. Calling the output a generated test would be
a lie, and the editor and README must call it a skeleton.

This feature is the first payoff of "capture everything": Phase 1 captured
ranked `locators` on every target and never read them. The skeleton reads them.

## Where it lives

Pure trace-to-string generation with no DOM dependency, so it lives in
`packages/trace` as `skeleton.ts` and is fully unit tested. The editor adds one
export button that writes the returned text to a `.spec.ts` download. No new
dependency, no renderer, no runtime that executes the skeleton.

## Input

The existing parsed `TraceEvent[]` and `RecordingMeta`. No schema change. No new
event type. The generator reads only fields already captured:

- `navigation` events and `meta.url` for `page.goto`
- `interaction.click` targets and their ranked `locators`
- `interaction.input` targets, their `locators`, and masked `value`
- `route` transitions between steps, for suggested assertions
- newly present target identities after a step, for suggested assertions

## Locator resolution

Each actionable event resolves to one Playwright accessor, chosen from the
highest-confidence locator the descriptor carries, mirroring the ranking already
defined in `packages/trace/locators`:

| Locator      | Playwright accessor                              |
|--------------|--------------------------------------------------|
| `testId`     | `page.getByTestId('create-report')`              |
| `role`+name  | `page.getByRole('button', { name: 'Create...' })`|
| `label`      | `page.getByLabel('Report name')`                 |
| `text`       | `page.getByText('Create report')`                |
| `css`        | `page.locator('main > header > button.primary')` |

If a target carries no locators, the step is emitted as a `// TODO: no locator
resolved for this step` comment with the recorded accessible name, never a
fabricated selector. Strings are emitted through a single-quote escaper so a
name containing a quote cannot break the generated source.

## Actions

- `interaction.click` -> `await <accessor>.click();`
- `interaction.input` -> `await <accessor>.fill('<value>');`
  Masked values (`[MASKED]`) are emitted verbatim as the recorded placeholder,
  because the real value was never captured. A comment marks it:
  `// value was masked at capture time`.
- `navigation` that changes route -> `await page.goto('<url>');` for the first
  load; later same-tab route changes become suggested assertions, not gotos.

## Suggested assertions, as comments only

Derived strictly from observed facts, never from intent:

- Route changed after a step:
  `// suggested: route changed to /reports/new`
- A target identity newly present in a later step that was absent before:
  `// suggested: text "Report created" became visible`

The recorder cannot observe successful responses in Phase 4 (network capture is
off by default, forever), so response assertions are not emitted. Every
suggestion is a comment beginning `// suggested:` so a reader cannot mistake it
for an executed assertion.

## Output shape

```ts
import { test, expect } from '@playwright/test';

// Generated by Cutscene from recording rec_01H8XK.
// This is a skeleton, not a test. Assertions are suggested as comments.
test('analytics-overview', async ({ page }) => {
  await page.goto('https://app.example.com/dashboard');
  await page.getByTestId('create-report').click();
  // suggested: route changed to /reports/new
  await page.getByRole('textbox', { name: 'Report name' }).fill('[MASKED]');
  // value was masked at capture time
});
```

The test title derives from the recording id. `expect` is imported so a user
can promote a suggestion to an assertion without editing imports.

## Editor control

One `SKELETON` action in the export surface: `Export Playwright skeleton`. It
serialises the current bundle's events and downloads `<recordingId>.spec.ts`.
It does not run FFmpeg and does not touch the video. Disabled when no bundle is
loaded. The button label says skeleton, not test.

## Failure behaviour

Generation is a pure function returning a string; it cannot fail on a valid
parsed bundle. A bundle with zero actionable events yields a valid skeleton
whose body is a single `// no actionable events captured` comment, so the file
is always well-formed TypeScript.

## Verification

- Unit-test each locator tier maps to the correct accessor.
- Unit-test click, fill, masked fill, first goto, and route-change suggestion.
- Unit-test string escaping for names containing quotes and backslashes.
- Unit-test the no-locator TODO path and the empty-flow comment body.
- Generate a skeleton from a real recorded bundle; report the emitted step
  count, the locator tier used per step, and confirm the output parses as
  TypeScript with `tsc --noEmit`.
