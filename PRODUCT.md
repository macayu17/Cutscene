# PRODUCT.md

## Register

brand

The surface in question is the public launch site under `site/`. The editor in
`packages/editor` is a product surface with its own rules; this file's default
covers the marketing site.

## Platform

web

## Users

**Primary:** an open source maintainer or a devrel engineer at a developer
tools startup. They ship a DOM-based web product frequently. They own the demo
GIF in the README and the video on the landing page, and both are perpetually
out of date. They live in GitHub, they understand CI, they will adopt an open
source tool with a rough edge, and they will not create an account to try
something.

**Context of use:** a browser tab opened from a link in a repo, a newsletter,
or a hackathon gallery. Skeptical, fast-scrolling, has seen a hundred screen
recorders.

## Purpose

Convince a technical visitor, in one scroll, that Cutscene records something no
other screen recorder records, and get them to touch the live interactive demo.

**Success:** the visitor opens the hosted demo and clicks through it.

## Positioning

The only screen recorder whose demos fail like a failing test when the product
changes, because it records the page's structure alongside the pixels.

## Brand personality

Precise. Measured. Unbluffing.

The voice of an instrument, not a salesperson. Every claim on the page is a
number that can be reproduced locally. The page states its limitations before
its features, exactly as the README does. Confidence comes from evidence, never
from adjectives.

## Primary CTA

**Try the live demo.** Opens the hosted interactive click-through.

**Secondary:** view the source on GitHub.

## Belief ladder

What the visitor must believe, in order:

1. Ordinary screen recordings only know pixels, and every editor built on them
   is guessing.
2. Cutscene records the DOM alongside the pixels, so it is not guessing.
3. That single capture pays out repeatedly: video, interactive guide, docs,
   screenshots, test skeleton, accessibility findings.
4. Because the recording knows the element, it can be replayed against a
   changed build, and it fails honestly when the product moves.
5. This is real and running, not a mockup.

## The one line to remember

A stale screen recording looks fine right up until someone notices. A stale
Cutscene demo fails like a failing test.

## Proof on hand

All measured locally and reproducible with `pnpm test && pnpm typecheck &&
pnpm build && pnpm e2e`:

- 10 of 10 sampled zooms landed on the correct element; mean timing error 0.258
  frame, worst case 0.422.
- Interactive export: 5 of 5 hotspots in order, maximum hotspot edge error
  0.014435 rendered pixels, zero console errors.
- The quality report found a real defect on a real site: all five checkboxes on
  the recorded TodoMVC path expose no accessible name.
- 315 tests, 5 of 5 packages typecheck clean, 6 of 6 Chromium E2E.

No testimonials, no logos, no user counts. The project has none, and inventing
them would contradict the brand.

## Anti-references

- Generic SaaS template: gradient blobs, three identical icon-and-heading
  cards, a centered stat row, purple-to-blue everything.
- Enterprise or corporate: stock team photos, navy and grey, trust badges, logo
  walls, pricing tiers.
- Overdesigned or loud: heavy 3D, glassmorphism, particle fields, colors that
  compete with the product screenshots.
- Fake terminals. The product is an instrument, not a shell prompt.
- A plain README dumped on a white page.

## Aesthetic lane

**Measurement instrument.** The visual language is the product's own: a
bounding box that locks onto a target, coordinate readouts, crop marks, a
clip-path wipe that reads as a measurement being taken. Not a terminal, not an
editorial magazine, not a dashboard.

The central motif is the box. It begins as a guess drifting around a cursor,
snaps onto a real element, multiplies into artifacts, becomes a row in a drift
report, fails, and is repaired.

## Accessibility

- WCAG AA contrast minimum on all text.
- Full keyboard operation with visible focus, matching the editor's existing
  `outline: 2px solid` treatment.
- `prefers-reduced-motion` removes pinning, scrubbing, and transforms while
  keeping every piece of content readable and in order.
- Semantic landmarks and a skip link. Content must never be gated behind a
  scroll-triggered class.

## Strategic design principles

1. **Show the recording, not a metaphor for it.** Imagery on this page is real
   product output: real editor frames, the real interactive player, real report
   text. No illustrations.
2. **Every number is reproducible.** If a figure cannot be regenerated by a
   command in the README, it does not go on the page.
3. **Limitations before features**, as in the README. Chrome only, DOM-based
   apps only, no cross-origin iframes, open shadow roots only.
4. **The box carries the narrative.** Sections transform the same motif rather
   than resetting to a new one.
